# ADR 0004: sora-player の UI を流用し、ローカル管理機能は持ち込まない

## Status

Accepted

## Context

`sora-gallery` は完全新規 UI ではなく、`sora-player` のガワ・雰囲気・操作感を Web 公開用に再構成する方針である。

ただし、`sora-player` にはローカル管理アプリとしての機能が UI と密結合している。

## Decision

流用するもの:

- 全画面ギャラリー風 UI。
- 黒背景、グラス UI、縦長カード、レスポンシブグリッド。
- 検索 UI、タグチップ、再生画面、prompt 表示。
- 日本語 IME 対応。
- 基本キーボード操作とスマホ縦スワイプ。
- Tailwind CSS ベースのスタイル。

持ち込まないもの:

- Finder 連携。
- ファイル名表示。
- 動画カードの選択チェック UI。
- 一括選択。
- ローカルタグ編集。
- サムネイルのサーバー保存。
- ローカルファイルシステムを前提にしたエラー表示。

## Consequences

- `sora-player` らしい操作感を維持できる。
- 公開サイトとして不要または危険なローカル管理機能を排除できる。
- `page.tsx` からの単純コピーではなく、表示ロジックを選別して再構成する必要がある。
